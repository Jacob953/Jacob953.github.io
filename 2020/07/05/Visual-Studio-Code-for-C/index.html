Visual Studio Code for C++

Posted on 2020-07-05 Edited on 2020-07-21

玩过VScode的小伙伴都知道，VScode是一款让人又恨又爱的编辑器

我也不出意外，失败了很多次，但还好结果是棒棒的

这篇博客将会将我的心得分享给大家

前提条件：

​ 1.Visual Studio Code 1.46.1.0

​ 2.MinGW

Windows系统下，VScode下载和配置C++环境

下载VScode的注意事项

​ VScode只需要去官网下载即可，下载技巧将会稍后更新。

下载MinGW的注意事项

选择Ming-w64的原因：

​ 目前关于配置VSCode配置环境比较常用的工具链有WinGW、WinGW-w64和TDM-GCC

​ 但据说目前在Windows下调试仅支持Cygwin和 MinGW，于是我们先排除掉TDM-GCC(尽管TDM-GCC是最新的稳定版本的gcc工具集:dog:)

​ 至于Cygwin，这是一个将Linux应用到Windows下的模拟Posix系统，而MinGW是用于开发Windows应用的开发环境，实际上，两者都提供了部分Linux的应用

​ 最后，我们将选择WinGW-w64，WinGW和WinGW-w64实际上不是同一个工具链，WinGW又称mingw32，MinGW-w64最初的确是MinGW的一个分支而诞生，但现在已经独立发展，这就不用多说了吧

下载Ming-w64的注意事项

​ 1.为了避免小白掉坑，这里直接传送门：

https://sourceforge.net/projects/mingw-w64/files/

​ 2.选择适合本机的版本

​ 进入传送门后，切忌直接点击Download Lastest Version（因为我们还没有选择具体的版本），一直向下滑，我们可以看到很多链接，选择对应的版本即可。

​ 如上文所说posix适用于Linux，因此选择win32，同时，sjlj相对seh是一种更加古老的版本，win64适配性能更好的seh，当然，如果是win32，亲，这边建议您选择dwarf，它的性能比sjlj更适用且只能用于win32

​ 下载好Mingw-w64 >> 我们直接解压就行

​ 但需要记得解压的位置:joy:

​ 进入解压后的文件夹，里面有个bin的文件夹，进去就会发现有很多编译器，我们后面将会调用，一级棒​:+1:

添加系统变量

​ 1.右键 此电脑 ，选择 属性

​ 2.选择 环境变量（N）…

​ 3.选择系统变量的 新建

​ 4.选择 mingw64 下的 bin 文件夹（配置环境应该找到本机对应的地址，切忌直接用我的地址，除非我俩的路径一摸一样）

D:\Complier\VSCode\mingw64\bin

​ 5.一定要记得 确认 退出

配置 VScode下的C++环境

注意事项：

​ 1.不管更改任何文件，确认无误后一定要记得保存

​ 2.配置环境是以本机为基础的，切忌直接复制粘贴，需要修改的内容我会一一具体指出

一、下载c++插件

​ 在扩展里输入c++，一般来说第一个就是

（自己的常用插件，后续慢慢更新）

​ 其他四个是我正在使用哦个的插件，第二个是主题，第三个是区块上色（和codeblocks很像，这也是我以前比较喜欢的编译器），第五个是汉化，大家都懂吧:dog:

真 · VScode下++环境配置

​ 1.我们先在本地建立一个测试文件夹 >> 在VSCode中打开 >> 新建一个测试文件

​ 右上角第一个就是建立一个text.cpp，注意添加后缀cpp

​ 2.接着再来一段最经典的Hello World

​ 2.保存 >> 按F5进行编译 >> 选择第一个（GDB） >> 配置launch.json

配置 launch.json

​ 1.复制我的代码覆盖一下源码，需要注意的是

"miDebuggerPath": "D:\\Complier\\VSCode\\mingw64\\bin\\gdb.exe"

​ 选择的是编译器的路径，因此需要对应到本机的编译器上

​ 2.保存 >> 按F5编译 >> 选择配置任务 >> 适用模板创建tasks.json文件 >> MSBuild >> 配置tasks.json

配置 tasks.json

​ 1.对比配置 launch.json将路径更改到与本机对应起来，同时，要注意：

"preLaunchTask": "g++"`和` "label": "g++" `的内容要完全一致

​ 2.保存 >> 回我们的text.cpp >> F5 >> 奇迹就出现啦:smile:
